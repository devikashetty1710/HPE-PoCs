ID: GH-101
Title: Special characters in passwords break the authentication endpoint
Description: Reproduction: create an account with password "P@ssw0rd!". Attempting to log in returns HTTP 500. The issue is in the auth middleware which does not URL-encode special characters before forwarding to the identity provider.

ID: GH-102
Title: Performance degradation with large user record counts
Description: Profiling shows that loading the main dashboard does a full table scan on the records table. For accounts with over 10k rows the query alone takes 25 seconds. Proposed fix: server-side pagination + indexed queries.

Title: WebSocket connection drops after 60 seconds of inactivity
